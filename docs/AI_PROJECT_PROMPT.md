# Photography Fill Light 项目 AI 提示词

更新时间：2026-03-20

## 1. 项目 review 摘要

### 1.1 当前项目的真实状态

- `program/` 里的 STM32 工程还在开发早期，目前基本是 CubeMX 生成的外设骨架，不应把现有固件当成“功能已实现”的参考。
- 真实硬件架构应优先以网表 `datasheet/Netlist_单色温_2026-03-15.tel`、`program/Fill_Light.ioc` 和器件 datasheet 为准。
- 根目录 `instruction.md` 明显混入了旧方案信息，出现了双通道、`MP3910A`、`IRF540N` 等与当前网表不一致的描述，使用前必须人工校对。

### 1.2 已确认的硬件架构

- 主控：`STM32F030F4P6TR`，封装 `TSSOP20`。
- 输入电源：项目目标是宽压输入补光灯，现有辅助电源链路按网表可确认为 `VIN -> +12V -> +5V -> +3.3V`。
- `U1 MP4420`：从 `VIN` 降压出 `+12V`，给运放和半桥驱动供电。
- `U3 MP4420`：从 `+12V` 再降压出 `+5V`。
- `U2 MP20051`：从 `+5V` 线性稳压出 `+3.3V` 供 MCU。
- Boost 主功率级：`U12 MP1907` 半桥栅极驱动器，驱动 `Q1/Q2`，配合 `U11 33uH` 电感构成升压级，输出网名为 `VOUT`。
- Boost 占空比语义约束：这个项目里真正有意义的 Boost 占空比定义在下管（low-side）导通时间上。若当前固件实现是以“上管 PWM”去生成互补驱动，则软件里的占空比必须做反相后再写比较寄存器，不能把目标 Boost 占空比直接当成上管占空比使用。
- LED 恒流级：`U7 TLV2372` + `Q5 IXTH30N50L2` + `R14 0.1R` 组成模拟线性恒流回路。
- MCU 调光方式：`DAC_CUR_CTR` 由 MCU PWM 经 RC 滤波后形成模拟参考，送入 TLV2372，调节线性恒流值。
- 采样链路：
  - `ADC_VOUT`：通过 `R15/R18` 分压采样升压输出电压。
  - `ADC_VDS`：通过 `R19/R20` 分压并配合钳位二极管采样线性恒流 MOS 的工作电压。
- 风扇控制：`FAN_CTRL` 通过 `Q3 AO3400A` 做低边开关控制风扇。
- 人机交互：`EC11_A`、`EC11_B`、`EC11_D` 直接接编码器，编码器公共端接地，没有看到外部上拉，软件侧应考虑内部上拉。

### 1.3 已确认的 MCU 引脚分配

- `PA0` -> `ADC_IN0`
- `PA1` -> `ADC_IN1`
- `PA2` -> `FAN_CTRL`
- `PA3` -> `EC11_A`
- `PA4` -> `EC11_B`
- `PA5` -> `EC11_D`
- `PA7` -> `TIM3_CH2`
- `PA9` -> `MP1907_EN`
- `PA10` -> `TIM1_CH3`
- `PB1` -> `TIM1_CH3N`
- `PF0/PF1` -> 外部 `8MHz` 晶振

### 1.4 当前固件存在的关键缺口

- 还没有任何实际控制框架：没有状态机、没有启动顺序、没有保护逻辑、没有参数管理。
- `TIM1` 目前只是配置成 `Output Compare`，没有形成 `MP1907` 所需的真实 PWM/互补驱动，也没有看到启动代码。
- `TIM3` 虽然挂在 `PA7`，但当前还没有 PWM 启动与占空比更新逻辑。
- `ADC` 只完成了通道初始化，没有采样调度、DMA、滤波、标定和控制闭环。
- `EC11` 目前只是普通输入口，没有 EXTI/轮询解码，也没有上拉配置。
- `FAN_CTRL` 目前只是普通 GPIO，且主循环里被持续拉低，不是温控 PWM。

### 1.5 AI 在本项目中应遵守的事实优先级

1. 先信网表和 `ioc`。
2. 再信器件 datasheet。
3. 再看当前源码里已经落地的部分。
4. `instruction.md` 只能当历史草稿，不能直接当真相。
5. 讨论 Boost 占空比时，默认指的是下管有效占空比；如果代码当前控制的是上管通道，就必须先做占空比反相再分析。

## 2. 可直接复制给 AI 的提示词

```md
你现在接手的是一个基于 STM32F030F4P6TR 的摄影补光灯项目。请先把下面这些内容当成项目的硬约束，再开始分析、写代码或 review。

### 项目目标

- 这是一个宽压输入的 COB 补光灯控制项目。
- 主控是 STM32F030F4P6TR。
- 系统目标是：通过 Boost 升压得到 LED 需要的输出电压，再通过线性恒流级实现稳定无频闪调光。
- 用户交互器件是 EC11 编码器，风扇由 `FAN_CTRL` 控制散热。

### 真实硬件架构

- 电源链路：`VIN -> MP4420(U1) -> +12V -> MP4420(U3) -> +5V -> MP20051(U2) -> +3.3V`
- Boost 功率级：`MP1907(U12)` 驱动 `Q1/Q2` 和 `33uH` 电感，输出网名是 `VOUT`
- 非常重要：这个项目里 Boost 的“有效占空比”是指下管导通占空比，不是上管占空比。如果固件当前通过上管 PWM 生成互补输出，那么软件计算出的目标占空比要先反相，再写入对应比较寄存器。
- 恒流级：`TLV2372(U7)` 驱动 `IXTH30N50L2(Q5)`，配合 `0.1R` 采样电阻 `R14` 构成模拟线性恒流
- MCU 通过 PWM+RC 的方式在 `DAC_CUR_CTR` 上实现伪 DAC，给 TLV2372 提供电流参考
- `ADC_VOUT` 用来采样 Boost 输出电压
- `ADC_VDS` 用来采样线性恒流 MOS 的工作电压/余量
- 风扇由 `FAN_CTRL` 经 `AO3400A(Q3)` 做低边开关控制
- EC11 的 `A/B/D` 直接进 MCU，编码器公共端接地，软件侧要考虑上拉

### MCU 引脚分配

- `PA0` -> `ADC_VDS`
- `PA1` -> `ADC_VOUT`
- `PA2` -> `FAN_CTRL`
- `PA3` -> `EC11_A`
- `PA4` -> `EC11_B`
- `PA5` -> `EC11_D`
- `PA7` -> `TIM3_CH2`，预期用于 `DAC_CUR_CTR`
- `PA9` -> `MP1907_EN`
- `PA10` -> `TIM1_CH3`
- `PB1` -> `TIM1_CH3N`
- `PF0/PF1` -> 外部 8MHz 晶振

### 当前软件状态

- `program/` 里的工程还没写完，目前是早期开发阶段。
- 现有代码大多只是 CubeMX 生成的初始化骨架。
- 不能假设 Boost 控制、恒流控制、EC11 解码、风扇控制、保护逻辑已经写好。
- 做任何修改前，要先区分“硬件已经定了”与“固件还没实现”。

### 你在这个项目里工作时的规则

1. 先阅读并交叉核对以下文件：
   - `datasheet/Netlist_单色温_2026-03-15.tel`

4. 你输出结论时，要明确区分：
   - 已由网表/源码确认的事实
   - 基于器件连接做出的合理推断
   - 还需要人工确认的点
5. 如果要实现控制算法，优先围绕以下模块拆分：
   - 上电与软启动

### 期望你在分析或写代码时重点关注的风险

- EC11 没有外部上拉时，MCU 端是否正确开启内部上拉
- 风扇口是否需要 PWM，而不是简单 GPIO
- TIM1 是否正确配置成适合 MP1907 的互补输出和安全时序
- Boost 占空比语义是否搞反：目标量应是下管占空比；若直接把目标占空比写成上管 PWM，会导致控制方向错误

- ADC 量程、分压比例、钳位网络和软件换算是否一致

- Boost 输出电压与线性恒流级余量之间是否存在联动控制策略
```

## 3. 建议的后续开发顺序

1. 先补齐引脚模式与底层驱动初始化，修正 EC11 上拉、风扇输出模式、TIM1/TIM3 的目标模式。
2. 再实现上电顺序、MP1907 使能、基础 PWM 输出和 ADC 读取闭环。
3. 最后再叠加 EC11 交互、亮度映射、风扇策略和故障保护。
