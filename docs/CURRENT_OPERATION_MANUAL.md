# Photography Fill Light Current Operation Manual

更新时间: 2026-03-21

## 1. 文档目的

这份手册不是代码说明，而是当前项目的落地操作顺序。

目标只有三个:

1. 先把测量搞准。
2. 再把 Boost 从“固定 Vout”改成“只给线性恒流级留最小余量”。
3. 最后再去追求满功率和效率。

后续开发时，优先以这份手册、网表 `datasheet/Netlist_单色温_2026-03-15.tel`、工程 `program/Fill_Light.ioc` 为准。

## 2. 当前已经确认的硬件事实

- 主控是 `STM32F030F4P6TR`。
- Boost 主功率级是 `MP1907 + Q1/Q2 + 33uH`，输出网名为 `VOUT`。
- 线性恒流级是 `TLV2372 + IXTH30N50L2(Q5) + R14(0.1 ohm)`。
- MCU 通过 `TIM3_CH2 -> RC -> DAC_CUR_CTR` 给运放参考，控制灯电流。
- `ADC_VOUT` 分压为 `150k / 10k`，软件换算关系是 `VOUT = VOUT_pin * 16`。
- `ADC_VDS` 分压为 `100k / 15k`，软件换算关系是 `ADC_VDS_net = VDS_pin * 115 / 15`。
- 已确认: 当前 `Vds` 调试语义按 `Q5 drain 对地电压` 理解，不再按 `drain-source` 直接理解。
- 风扇由 `FAN_CTRL -> Q3(AO3400A)` 控制。
- EC11 已用于按键开关机和旋钮调节亮度。

## 3. 当前必须牢记的控制约束

### 3.1 Boost 占空比语义

- 本项目里有意义的 Boost 占空比，是“下管有效占空比”。
- 当前 PWM 硬件比较值落在上管通道，所以软件写 TIM 比较值时必须先反相。
- 讨论占空比时，默认都按下管有效占空比理解。

### 3.2 36.42V 不是必须死控的总线电压

- `36.42V` 更接近 COB 在额定电流附近的一个工作点。
- 它不是 Boost 母线必须永远锁死的唯一目标。
- 真正应该满足的是:

`Vout_bus = Vled_actual + 线性恒流级最小工作余量`

如果把 `Vout` 长时间定得过高，额外的压差都会烧在 `Q5` 上。

### 3.3 关于 `ADC_VDS` 的当前结论

这件事已经确认:

`Vdrain_to_gnd`

在这个前提下，可得到:

- `Vled_est = Vout - Vdrain_to_gnd`
- `Vsource_est = Iled * 0.1 ohm`
- `Vds_true_est = Vdrain_to_gnd - Vsource_est`

这件事要先确认，因为它会直接决定后面闭环目标到底控什么。

## 4. 当前固件的工作基线

- ADC 工程量已经按 `mV` 输出。
- `boost_voltage_debug` 里已经有:
  - `vout_mv`
  - `vds_mv`
  - `vout_minus_vds_mv`
- `EC11` 旋转方向已反相。
- 亮度 DAC 默认占空比已改为 `0.05`，即 `50 permille`。
- 状态机已经有 `SYS_STANDBY -> SYS_SOFT_START -> SYS_RUNNING`。
- 风扇在 `SOFT_START` 和 `RUNNING` 状态下已开启。
- 当前 Boost 已切到“固定余量版本”。
- 当前实现是用 `vds_mv_lpf` 作为 `Q5 drain-to-gnd` 余量反馈，默认目标为 `1000mV`。
- `vout_ctrl_ref_mv` 现在是随 `Vled_est + headroom_target` 实时变化的动态母线目标，便于在 debug 里直接看。

## 5. 你接下来应该一项一项做的事情

## Step 1. `ADC_VDS` 物理意义已确认

结论:
当前 `ADC_VDS` 就按 `Q5 Drain -> GND` 理解。

操作:

1. 上电后让系统工作在低亮度、低风险状态。
2. 用万用表测 `Q5 Drain -> GND`。
3. 再测 `Q5 Source -> GND`。
4. 再看软件里的 `vds_mv`。
5. 比较 `vds_mv` 更接近哪一个。

后续约束:

- 后续控制目标应围绕 `Vdrain_to_gnd` 设计。
- 不再把当前 `vds_mv` 直接当成 `Drain -> Source`。

备注:

- 没确认这一步之前，不要把“Vds 目标值”写死进控制逻辑。

## Step 2. 把 ADC 测量先做准

目的:
避免闭环参考建立在错误测量上，导致 `Vout` 超过灯额定工作点太多，或者压差不够导致恒流掉线。

操作:

1. 用万用表实测 `VOUT`。
2. 对比当前软件里的 `vout_mv`。
3. 用万用表实测 `Q5 Drain -> GND` 或确认后的实际采样点。
4. 对比当前软件里的 `vds_mv`。
5. 至少做一次比例校准，最好做两点校准。
6. 后续 ADC 进入闭环前再做均值/IIR 滤波。

建议:

- 先做比例修正，再考虑 offset。
- 不要直接相信 `3.3V` 基准一定准。
- 高阻分压下，采样时间过短也会让读数偏低。

完成标准:

- `VOUT` 读数与万用表误差进入你能接受的范围。
- `VDS` 读数与万用表误差进入你能接受的范围。
- 误差水平明确记录下来，不再靠感觉估算。

## Step 3. 先定一个绝对过压保护上限

目的:
不管后面闭环怎么写，先保证不会把 COB、输出电容和功率级顶到危险区域。

操作:

1. 查清 `VOUT` 端所有器件的耐压:
   - `C18`
   - `C19`
   - `C20`
   - `C22`
   - `C36`
   - COB 接口相关器件
2. 选出其中最低的耐压器件。
3. 以该最低耐压为基准，留足裕量后定义 `Vout_abs_max`。

逻辑:

- `Vout_abs_max` 是独立保护。
- 它不依赖亮度、不依赖电流、不依赖正常闭环。
- 一旦超过，就应该降占空比或关断功率级。

完成标准:

- 你能明确说出“这块板子 VOUT 软件绝对不允许超过多少伏”。

当前固件设置:

- `Vout_abs_max = 42000mV`

## Step 4. 不要再把运行态主目标写成固定 `36.42V`

目的:
降低 `Q5` 在线性区上的无谓耗散。

正确思路:

- 亮度旋钮决定 `DAC_CUR_CTR`，也就是目标电流。
- 模拟恒流环负责快速把电流拉到目标。
- 数字 Boost 环负责慢速调节 `VOUT`，让 `Q5` 只保留最小必要工作余量。

运行态目标应该变成:

`Vout_target_dynamic = Vled_actual + headroom_target`

而不是:

`Vout_target_fixed = 36.42V`

如果当前 `ADC_VDS` 实际采的是漏极对地电压，那么更适合控的是:

`Vdrain_to_gnd_target`

## Step 5. 先做“固定余量版本”，不要一步上最复杂

当前状态:

- 这一步已经落地了第一版代码。
- 软启动阶段拉升的是 `vds_ctrl_ref_mv`，运行阶段维持 `vds_target_mv = 1000mV`。
- 后续工作重点不再是“有没有固定余量控制”，而是“1000mV 在高亮和热机下是否仍然够用，需要靠实测确认”。

目的:
先把控制方向做对，再做精细化。

第一版建议:

- 先不让余量目标跟电流联动。
- 先给一个保守的固定目标余量。
- 如果余量太大，Boost 降一点。
- 如果余量太小，Boost 升一点。

原因:

- 你现在还没有独立的 MCU 电流闭环采样。
- 亮度参考来自 PWM+RC，电流是模拟环在控。
- 先做固定余量版本，最容易验证方向和稳定性。

完成标准:

- 亮度变化时，`Q5` 压降不会一直飙高。
- 降亮度后，Boost 会自动跟着往下收。
- 提亮度后，Boost 会自动补一点余量，恒流不掉线。

## Step 6. 再做“余量随电流变化版本”

目的:
进一步压低损耗，提高效率和热表现。

建议逻辑:

- `headroom_target` 不要恒定。
- 电流越大，允许的目标余量可以略大一点。
- 电流越小，目标余量应更低。

如果 `ADC_VDS` 是 `Vdrain_to_gnd`，则更合理的目标是:

`Vdrain_target = Vsource_est + Vds_margin_target`

其中:

- `Vsource_est = Iled * 0.1 ohm`
- `Vds_margin_target` 是 Q5 保持线性恒流所需的最小安全余量

注意:

- 这一步必须建立在 Step 1 和 Step 2 已经确认完成的前提下。
- 否则目标公式虽然好看，实物上会跑偏。

## Step 7. 用实验找出最小稳定余量，不要靠猜

目的:
确定 `Q5` 在线性恒流区真正需要的最小压差。

操作:

1. 固定一个亮度档位。
2. 缓慢降低 Boost 输出。
3. 观察什么时候亮度开始掉、恒流开始失稳、纹波开始明显。
4. 记录此时的:
   - `VOUT`
   - `Drain -> GND`
   - `Source -> GND`
5. 给这个临界点再留一点安全余量。

你最终要得到的是:

- 低亮度下的最小稳定余量
- 中亮度下的最小稳定余量
- 高亮度下的最小稳定余量
- 热机状态下的最小稳定余量

完成标准:

- 余量目标由实测决定，而不是拍脑袋给一个常数。

## Step 8. 满功率时，VOUT 本来就应该略高于 LED 实际工作电压

目的:
避免把“略高于额定电压”误判为错误。

正确理解:

- 想跑满功率，`VOUT` 必须高于 COB 当时的实际 `Vf`。
- 多出来的这部分，就是线性恒流级的工作余量。
- 所以满功率时 `VOUT` 略高于 `36.42V` 是正常的。

真正要避免的是:

- `VOUT` 高很多，但这些电压都白白烧在 `Q5` 上。

功耗关系:

`Pq5 = Vds_true * Iled`

所以多出来的每一点压差，都会直接变成热。

## Step 9. 后面要补的保护逻辑

这些保护建议放在“基本闭环跑通”之后再加，但必须规划进去:

- `VOUT` 绝对过压保护
- 余量不足保护
- 开路负载保护
- 异常高温保护
- ADC 异常值保护
- 软启动超时保护

尤其是开路负载场景:

- 如果 COB 断开，Boost 可能会一直往上顶。
- 这时不能再依赖正常的“最小压差控制”。
- 必须由绝对过压保护接管。

## 6. 建议的实施顺序

严格按下面顺序推进，不要跳步:

1. 先确认 `ADC_VDS` 真实物理意义。
2. 再校准 `VOUT` 和 `VDS` 采样。
3. 再确认 `VOUT` 绝对过压上限。
4. 再做固定余量版本控制。
5. 再做实测最小稳定余量。
6. 再做余量随电流变化版本。
7. 最后补全保护逻辑。

## 7. 当前阶段不要犯的错误

- 不要把 `36.42V` 当成唯一固定闭环目标。
- 不要在 ADC 未校准前就把保护门限贴得很死。
- 不要在未确认 `ADC_VDS` 含义前就写“Vds 控制公式”。
- 不要让正常运行完全依赖线性 MOS 吃掉大量压差。
- 不要只看亮不亮，还要看 `Q5` 发热和风扇负担。
